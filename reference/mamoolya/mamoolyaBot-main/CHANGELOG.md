# История изменений

## [Unreleased]

## [2.3.26] - 2025-10-28

### Fixes
- Streaming edits for `/fact` now ignore Telegram's "message is not modified" errors to prevent failures when the text stays the same between updates.

## [2.3.25] - 2025-10-28

### Fixes
- Groq sammaries retry up to `GROQ_MODEL_ATTEMPTS` per model with exponential backoff, and automatically fall back to GPT when available.
- Weekly summary skips the "топ ноулайферов" digest if AI generation fails, so users only see stats after a successful саммари.

## [2.3.24] - 2025-10-28

### Fixes
- Added Groq rate-limit aware fallback chain (e.g. switching to `llama-3.1-8b-instant`) so /summary keeps working even when the primary model hits TPM limits.

## [2.3.23] - 2025-10-28

### Fixes
- Automatically remap deprecated Groq model IDs (e.g. `llama3-70b-8192`) to their recommended replacements so existing deployments keep working after platform updates.

## [2.3.22] - 2025-10-28

### Fixes
- Updated Groq default model to `llama-3.3-70b-versatile` to align with the latest platform deprecations.

## [2.3.21] - 2025-10-28

### Fixes
- Groq availability check now reads `GROQ_API_KEY`/`GROQ_MODEL` from the runtime environment, so admin menu immediately reflects server-side updates without restart.

## [2.3.20] - 2025-10-28

### Fixes
- Groq integration no longer depends on the requests package; urllib is used instead.
- Menu hint now mentions requests if the module is missing.

## [2.3.19] - 2025-10-28

### Новое
- Добавлена полноценная поддержка Groq API: бот может использовать модель LLaMA3 (через Groq) для саммари и fallback.
- В меню настроек AI и в лаунчере появились кнопки для переключения на Groq.
- Саммари автоматически переключается на Groq, если OpenAI недоступен или квота исчерпана.

## [2.3.18] - 2025-10-28

### Новое
- В launcher переключение модели саммари больше не требует python-dotenv на сервере; настройки читаются напрямую из `.env`, а результат выводится понятным сообщением.

## [2.3.17] - 2025-10-28

### Новое
- В меню настроек AI появилась отдельная кнопка для выбора модели OpenAI GPT-3.5 Turbo.
- В launcher появились команды для переключения модели саммари на сервере между дефолтной GPT и GPT-3.5.
- Обновлён вызов OpenAI: можно указать конкретную модель для саммари без влияния на общий дефолт.

## [2.3.16] - 2025-10-28

### Новое
- Обработали APIError без status_code, теперь /summary не падает с сообщением "'APIError' object has no attribute 'status_code'".

## [2.3.13] - 2025-10-28

### Новое
- Через launcher (manage_bot.bat) можно смотреть и менять `MAMULYA_DIALOGUE_STYLE`, `MAMULYA_SUMMARY_TONE` и `MAMULYA_CONTEXT_DEPTH`.

### Улучшено
- Промпты Мамули теперь учитывают настроенный тон, читают ограниченное число свежих сообщений и не придумывают факты.

- Пополнены заготовленные ответы: приветствия, «психолог», предсказатель, гороскоп, игра в бутылочку и мемные фразы получили больше вариантов.

## [2.3.11] - 2025-10-27

### Исправлено
- Финальное сообщение после потоковой генерации больше не перезаписывается промежуточными обновлениями, поэтому гороскопы показываются с заголовком `🔮`.
- Команда `/webapp` теперь выдаёт валидный URL: при отсутствии настроек подключается дефолтный хост и подсказывает, как задать `STATIC_WEB_URL`.
### Изменено
- `/fact` запускается с текстом «Готовлю точный факт…», а новый промпт и параметры (`temperature=0.95`, `top_p=0.9`, `max_output_tokens=140`) делают ответ короче и точнее.
- В админке кнопка переименована в «📆 Итоги недели», из соц-меню убрана дублирующая «Предложка», а в настройках AI отображается актуальная модель OpenAI (`gpt-4.1-mini`).

## [2.3.9] - 2025-10-27
