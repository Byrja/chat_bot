## Структура проекта

Проект содержит следующие основные файлы:
- `bot.py` - основной файл бота
- `mamoolyaBot` - основной код. В `bot.py` осталась только точка входа.
  - `mamoolyaBot.bot.mamoolyaMain` старт бота теперь тут
- `.env` - секреты (ключи к апи)
- `requirements.txt` - список зависимостей

## OpenAI Configuration

- Default model is provided via `OPENAI_MODEL` (default `gpt-4.1-mini`).
- Configure fallback sequence with `OPENAI_FALLBACKS`, e.g. `gpt-4o-mini,gpt-3.5-turbo`.
- Responses now use the OpenAI Responses API with streaming updates.
- `OPENAI_TIMEOUT_MS` defines per-request timeout in milliseconds.
- Optional `PROMPT_CACHE_KEY` lets you reuse cached system prompts.
- `MAMULYA_DIALOGUE_STYLE` controls the default tone for live replies (`auto`, `kind`, `rude`).
- `MAMULYA_SUMMARY_TONE` tunes how дерзкими будут саммари (`mellow`, `balanced`, `spicy`).
- `MAMULYA_CONTEXT_DEPTH` sets how many последних сообщений попадает в промпт (3-50, default 6).
- `manage_bot.bat` теперь умеет показывать локальный `git status` и запускать «commit+push» одной командой (пункты 21-22).
- `Makefile` - скрипты для обновления и управления ботом
- `check_anon_messages.py` - скрипт для проверки анонимных сообщений в базе данных
- `publish_anon_messages.py` - скрипт для ручной публикации анонимных сообщений
- `*.bat` - батники для управления ботом на сервере Windows

## Игра в бутылочку

Бот теперь поддерживает игру в бутылочку! Чтобы начать игру, используйте команду [/bottle](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L1855-L1867).
Бот предложит участникам нажать кнопку "Кто играет?", после чего случайным образом выберет действие между двумя участниками.
Бот будет тегать пользователей через "@" в сообщениях.

### Примеры действий:
- @A говорит @B три честных комплимента (без иронии).
- @A записывает 10-секундный «танец» голосовым/видео для @B.
- @A кидает трек, который ассоциируется с @B, и объясняет в 1 предложении — почему.
- @A делает мем с @B (картинка + подпись) и кидает в чат.
- @A выполняет мини-поручение от @B в одну фразу
- @A присылает фото/картинку предмета, ассоциирующегося с @B
- @A отправляет "эмодзи-пантомиму" из 5 смайлов, а @B пытается угадать, что зашифровано.
- @A делится одним личным лайфхаком специально для @B

## Цитатник

Бот теперь поддерживает функцию "Цитатник" для сохранения и получения "золотых фраз" участников чата.

### Команды:
- [/quote](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L179-L195) - Сохранить сообщение как цитату (в ответ на сообщение)
- [/quotes](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L197-L224) - Получить случайную цитату с ссылкой на оригинал

### Использование:
1. Чтобы сохранить цитату, ответьте на нужное сообщение командой [/quote](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L179-L195)
2. Чтобы получить случайную цитату, используйте команду [/quotes](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L197-L224)

Каждая цитата сохраняется с ссылкой на оригинальное сообщение, что позволяет легко перейти к контексту.

### Веб-приложение:
Теперь цитаты также доступны через WebApp бота:
1. Откройте WebApp через команду [/webapp](file:///D:/Users/br/Documents/GitHub/mamoolyaBot/mamoolyaBot/bot.py#L771-L783) в Telegram
2. Нажмите кнопку "Цитатник" на главной странице
3. Просматривайте цитаты и переходите к оригинальным сообщениям

## Обновление

1. SSH
2. Cделай:
   ```bash
   cd /root/tg_bot_git
   make update
   ```

При выполнении команды `make update` произойдет следующее:
- Активация вируального окружения
- Получение последних изменений из git-репозитория
- Установка всех зависимостей из файла requirements.txt
- Перезапуск systemd-сервиса tg_bot

Код лежит в `Makefile`.
Виртуальное окружение ожидается в папке `venv` в корне

## Утилиты для работы с анонимными сообщениями

### Проверка анонимных сообщений
Для проверки содержимого таблицы анонимных сообщений в базе данных можно использовать скрипт:
```bash
python check_anon_messages.py
```

### Ручная публикация анонимных сообщений
Для ручной публикации неопубликованных анонимных сообщений можно использовать скрипт:
```bash
python publish_anon_messages.py
```

## Система анонимных сообщений

Анонимные сообщения теперь публикуются немедленно после отправки через веб-интерфейс. Каждое сообщение публикуется только один раз.

Ранее система публиковала сообщения каждые 5 минут, но теперь это изменено для более отзывчивого пользовательского опыта.

## Устранение неполадок

Если анонимные сообщения не публикуются, следуйте инструкциям в файле [ANON_MESSAGES_TROUBLESHOOTING.md](file://d:\Users\br\Documents\GitHub\mamoolyaBot\ANON_MESSAGES_TROUBLESHOOTING.md).

## Примечание

Ранее анонимные сообщения не публиковались в группе из-за проблем с базой данных и несоответствия типов данных. Проблема была решена:
- Исправлено создание таблиц в базе данных
- Исправлено несоответствие типов данных поля user_id
- Настроена правильная работа автоматической публикации каждые 5 минут

Подробнее о внесенных изменениях и тестировании можно прочитать в файле ANON_MESSAGES_README.md.
